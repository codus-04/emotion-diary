개발자 관점의 문제 해결 기록

🔧 Expo SDK 버전 충돌

문제: SDK 56 설치 시 react-native-reanimated peer dependency 에러
해결: SDK 54로 다운그레이드 + npm install --legacy-peer-deps
교훈: SDK 버전 호환성을 먼저 확인 후 의존성 설치

🧭 expo-router → useState 전환

문제: expo-router가 SDK 54에서 navigation 충돌 발생
해결: App.tsx에서 useState로 screen 전환하는 단순 구조로 교체
교훈: 의존성 최소화 → 안정성 향상

📡 Expo Go 네트워크 연결

문제: QR 스캔 후 연결 안 됨 (방화벽 차단)
해결: Mac 방화벽 비활성화 + 동일 Wi-Fi 확인 / --tunnel 모드
교훈: 모바일 개발 환경 설정의 중요성

개발 환경 & 빌드 / 배포

GitHub 설치 가이드 포함
🛠️개발 환경 설정
① git clone + cd emotion-diary/emotion-diary
② npm install --legacy-peer-deps
③ cp .env.example .env → API 키 입력
④ Firebase 프로젝트 설정
⑤ npx expo start → QR 스캔
⚠ --legacy-peer-deps 필수 (SDK 54 호환성)
🚀빌드 & 배포 단계
DEBUG개발 중 실시간
npx expo start
PREVIEWAPK 사이드로드
npx eas build --profile preview
PRODUCTION실배포
npx eas build --profile production
빌드: 소스코드 → 실행 가능한 바이너리(.apk/.ipa)
배포: 사용자 기기에 전달하는 과정

ADR & 코드 품질 / 테스트

의사결정 기록 + 품질 관리
ADR-0001React Native + Expo 선택
결정: React Native + Expo SDK 54
이유: 웹 지식 재사용, Expo Go 즉시 테스트, 커뮤니티 풍부
ADR-0002Zustand 상태관리 선택
결정: Zustand (vs Redux, Context API)
이유: 보일러플레이트 없음, 학습 곡선 낮음, TypeScript 지원
ADR-0003Firebase 백엔드 선택
결정: Firebase Auth + Firestore
이유: Auth+DB 통합 제공, 무료 티어, 설정 간편
