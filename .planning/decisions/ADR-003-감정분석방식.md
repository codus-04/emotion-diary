# ADR-003. 감정 분석 처리 방식 선택

## 상태
accepted

## 작성일
2026-05-04

---

## 배경
일기 텍스트의 감정 분석을 어느 시점에,
어떤 방식으로 처리할지 결정해야 한다.
분석 타이밍, 데이터 저장 위치, 개인정보 보호를
종합적으로 고려해야 한다.

---

## 선택지

### Option A. 서버 기반 실시간 분석 (선택)
- 일기 저장 시 Claude API 서버로 전송해 분석
- 분석 결과를 로컬 DB(Isar)에 저장
- 인터넷 연결 필요

### Option B. 온디바이스 분석
- ML Kit / 경량 모델로 기기 내에서 분석
- 인터넷 없이 동작 가능
- 감정 뉘앙스 분석 품질 낮음

### Option C. 배치 분석 (야간 일괄 처리)
- 하루치 일기를 모아 야간에 일괄 분석
- API 비용 절감 가능
- 실시간 피드백 불가 → UX 저하

### Option D. 하이브리드 (온디바이스 + 서버)
- 간단한 감정은 온디바이스, 복잡한 감정은 서버 분석
- 구현 복잡도 높음
- 6주 일정 내 구현 부담

---

## 결정
**서버 기반 실시간 분석 (Option A)** 을 선택한다.

---

## 근거
1. **분석 품질 우선** — 복합 감정 및 한국어 뉘앙스는 Claude API가 압도적으로 우수
2. **즉각적 피드백** — 일기 저장 직후 감정 태그 확인 가능 → UX 향상
3. **구현 단순성** — 6주 일정 내 빠른 구현 가능
4. **개인정보 보호** — 일기 원문은 API 전송 후 로컬에만 저장, 서버에 보관하지 않음
5. **비용 관리** — Claude Haiku 사용으로 일기 1건당 약 $0.001 수준

---

## 데이터 흐름

사용자 일기 작성
↓
로컬 DB (Isar) 저장     ← 일기 원문 영구 저장
↓
Claude API 호출         ← 감정 분석 요청 (일기 텍스트 전송)
↓
감정 분석 결과 수신     ← primary_emotion, score, summary
↓
로컬 DB 업데이트        ← 감정 태그 저장
↓
UI 업데이트             ← 감정 태그 + 그래프 반영

---

## 개인정보 보호 정책
- 일기 원문은 분석 목적으로만 API에 전송
- Claude API는 입력 데이터를 학습에 사용하지 않음 (Anthropic 정책)
- 분석 완료 후 서버에 일기 데이터 미보관
- 로컬 DB 암호화 저장으로 기기 분실 시 보호

---

## 오프라인 대응 전략
- 인터넷 미연결 시 일기 작성 및 로컬 저장은 정상 동작
- 감정 분석은 "분석 대기 중" 상태로 표시
- 인터넷 연결 복구 시 자동으로 분석 재시도

---

## 결과 및 트레이드오프
- ✅ 높은 감정 분석 품질
- ✅ 즉각적인 피드백으로 UX 향상
- ✅ 구현 단순성으로 6주 일정 적합
- ⚠️ 인터넷 연결 필요 (오프라인 분석 불가)
- ⚠️ API 비용 발생 (일기 1건당 약 $0.001)
