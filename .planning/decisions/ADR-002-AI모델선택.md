# ADR-002. AI 감정 분석 모델 선택

## 상태
accepted

## 작성일
2026-05-04

---

## 배경
일기 텍스트에서 감정을 분석하고 태깅하는 AI 모델이 필요하다.
응답 속도, 비용, 감정 뉘앙스 분석 품질을 기준으로
적합한 LLM API를 선택해야 한다.

---

## 선택지

### Option A. Claude API - claude-haiku (선택)
- Anthropic이 만든 경량 고속 모델
- 감정의 뉘앙스와 맥락 이해 능력 우수
- 응답 속도 빠름 (haiku 기준 평균 1~2초)
- 비용 저렴 (input $0.25 / output $1.25 per 1M tokens)
- 한국어 감정 표현 분석 품질 높음

### Option B. GPT-4o (OpenAI)
- 범용성 최고, 멀티모달 지원
- 비용이 Claude Haiku 대비 약 10배 높음
- 감정 분석 특화 기능 없음
- 응답 속도 상대적으로 느림

### Option C. Gemini Flash (Google)
- 빠른 응답 속도
- 한국어 감정 분석 품질 Claude 대비 낮음
- Google 서비스 의존도 높아짐

### Option D. 온디바이스 모델 (ML Kit)
- 인터넷 없이 동작 가능
- 감정 분석 정확도 LLM 대비 낮음
- 복잡한 감정 뉘앙스 파악 어려움

---

## 결정
**Claude API (claude-haiku-4-5)** 를 선택한다.

---

## 근거
1. **감정 뉘앙스 분석 품질** — 단순 긍/부정이 아닌 복합 감정(예: 기쁘지만 불안한) 파악 가능
2. **한국어 성능** — 한국어 감정 표현에 대한 이해도 높음
3. **비용 효율** — GPT-4o 대비 저렴해 6주 개발 기간 내 비용 부담 낮음
4. **응답 속도** — Haiku 모델 기준 평균 1~2초로 UX에 무리 없음
5. **긴 컨텍스트** — 일기 전체 내용을 한 번에 분석 가능

---

## 프롬프트 설계 방향
당신은 감정 분석 전문가입니다.
아래 일기 내용을 읽고 다음을 JSON으로 반환하세요:

primary_emotion: 주요 감정 (기쁨/슬픔/분노/불안/평온/설렘 중 1개)
emotions: 감지된 감정 목록 (최대 3개)
score: 긍정 점수 (0~100)
summary: 감정 요약 한 줄 (20자 이내)


---

## 결과 및 트레이드오프
- ✅ 복합 감정 및 한국어 뉘앙스 분석 품질 우수
- ✅ 빠른 응답 속도로 UX 저하 없음
- ✅ 저렴한 비용으로 프로토타입 개발 적합
- ⚠️ 인터넷 연결 필요 (오프라인 분석 불가)
- ⚠️ API Key 보안 관리 필요 (.env 파일로 관리)
