# ADR-001. 모바일 플랫폼 프레임워크 선택

## 상태
accepted

## 작성일
2026-05-04

---

## 배경
AI 감정일기 앱을 iOS와 Android 양쪽에서 동작하도록 개발해야 한다.
제한된 개발 기간(6주) 안에 두 플랫폼을 모두 지원해야 하므로
크로스플랫폼 프레임워크 도입을 검토했다.

---

## 선택지

### Option A. Flutter (선택)
- Google이 만든 크로스플랫폼 프레임워크
- Dart 언어 사용
- iOS / Android 단일 코드베이스로 개발 가능
- ML Kit, flutter_speech_to_text 등 공식 패키지 풍부
- 핫 리로드로 빠른 개발 속도

### Option B. React Native
- Meta가 만든 크로스플랫폼 프레임워크
- JavaScript / TypeScript 사용
- 네이티브 브릿지 방식으로 성능 이슈 가능성
- 패키지 의존성 충돌 문제 빈번

### Option C. 네이티브 (Swift + Kotlin)
- iOS: Swift, Android: Kotlin 별도 개발
- 최고의 성능과 네이티브 UX 제공
- 두 플랫폼 별도 개발로 6주 일정 내 완성 불가능

---

## 결정
**Flutter** 를 선택한다.

---

## 근거
1. **단일 코드베이스** — iOS/Android를 하나의 코드로 개발 가능해 6주 일정에 적합
2. **ML Kit 공식 지원** — 음성 인식(flutter_speech_to_text) 등 필요 패키지 공식 지원
3. **빠른 개발 속도** — 핫 리로드로 UI 수정 즉시 확인 가능
4. **성능** — React Native 대비 자체 렌더링 엔진(Skia)으로 성능 우수
5. **생태계** — fl_chart, Isar, Riverpod 등 필요한 패키지 모두 Flutter 지원

---

## 결과 및 트레이드오프
- ✅ iOS / Android 동시 개발 가능
- ✅ 6주 일정 내 완성 가능한 개발 속도
- ⚠️ Dart 언어 학습 필요 (JavaScript 대비 생소할 수 있음)
- ⚠️ 네이티브 앱 대비 앱 용량이 다소 큼
