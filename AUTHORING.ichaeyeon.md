## 1. 나의 AI 협업 플로우

### 문서 작성 플로우

작성할 문서의 목적과 포함할 내용을 먼저 정리
↓
Claude에게 초안 요청
("AI 감정일기 앱의 {문서명} 작성해줘")
↓
초안을 읽으면서 프로젝트 상황에 맞게 직접 수정
↓
VS Code에서 파일에 붙여넣기 후 저장
↓
커밋 메시지에 AI 활용 여부 명시 후 push


### 코드 작성 플로우

구현할 기능의 요구사항을 먼저 정리
↓
Claude에게 코드 초안 요청
(아키텍처, 사용 패키지 명시)
↓
생성된 코드를 읽으면서 이해
(모르는 부분은 Claude에게 설명 요청)
↓
VS Code에 붙여넣기 후 직접 실행
↓
오류 발생 시 오류 메시지를 Claude에게 전달
↓
동작 확인 후 커밋


### 오류 해결 플로우

오류 메시지 전체 복사
↓
Claude에게 원인 및 해결 방법 질문
↓
제시한 해결 방법 적용 후 재실행
↓
해결되면 커밋 / 안되면 2번으로 반복

## 3. AI 결과물 검토 기준

### 문서 검토 체크리스트
✅ 프로젝트 이름이 올바른가? (AI 감정일기)
✅ 기술 스택이 맞는가? (Flutter, Claude API 등)
✅ 날짜가 올바른가?
✅ 다른 문서와 내용이 일치하는가?
✅ 빈 섹션 없이 모두 채워져 있는가?

### 코드 검토 체크리스트
✅ 실제로 실행했는가?
✅ 의도한 대로 동작하는가?
✅ 코드 내용을 이해하고 설명할 수 있는가?
✅ 에러 처리가 되어 있는가?
✅ .env에 있어야 할 Key가 코드에 하드코딩되지 않았는가?

## 4. 삽질 기록

### 2026-05-04
- **문제:** `git push` 시 한글 파일명이 깨져서 표시됨
- **원인:** 터미널 인코딩 문제
- **해결:** `git add .` 로 전체 추가 후 커밋

- **문제:** `sudo gem install cocoapods` 가 멈춤
- **원인:** 비밀번호 입력 대기 중인 상태
- **해결:** `brew install cocoapods` 로 대체 설치

## 5. 배운 것 기록

### Flutter
- `flutter doctor` 로 개발 환경 상태 확인 가능
- `flutter create app` 으로 기본 프로젝트 생성
- `code .` 으로 현재 폴더를 VS Code로 바로 열기 가능

### Git
- 파일마다 커밋을 쪼개면 변경 이력 추적이 쉬움
- `git add .` 으로 변경된 파일 전체 한 번에 추가 가능
- 커밋 메시지에 `docs:`, `feat:`, `fix:` 접두사 사용

### AI 협업
- Claude 초안을 그대로 쓰지 않고 반드시 수정하기
- 모르는 코드는 Claude에게 설명 요청하면 바로 이해 가능
- 오류 메시지 전체를 복붙하면 정확한 해결책을 받을 수 있음
